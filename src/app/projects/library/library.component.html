Project Overview

Academic project developed for two complementary courses at ISEP, aiming to build a comprehensive distributed library management system. The project demonstrates mastery of modern software architecture patterns, including microservices, CQRS, and event-driven design with RabbitMQ. Full API documentation is available via Swagger.

The system was designed to handle complex library operations including book management, user authentication, loan tracking, and genre categorization, all while maintaining data consistency across multiple services.

The frontend was developed for mobile platforms using Java for Android and React Native for cross-platform accessibility.

Challenges & Solutions

Distributed Data Consistency

Challenge: Maintaining data consistency across multiple microservices with separate databases.

Solution: Implemented event sourcing with RabbitMQ and eventual consistency patterns, along with saga orchestration for distributed transactions.

Concurrent Loan Operations

Challenge: Handling simultaneous loan requests for the same book without overbooking.

Solution: Implemented optimistic locking with version control and queue-based processing for loan requests.

Cross-Platform Compatibility

Challenge: Providing consistent user experience across Android and React platforms.

Solution: Designed RESTful API with comprehensive documentation and shared UI/UX patterns between platforms.
